Y原创 | Grasshopper中的日照分析插件——Sunflower

发表于 2020-07-06 | 分类于 Y原创 |

写在前面

时隔半个月，才抽出时间来更新。最近的项目实在是太恼火，由于项目太急，只能是在方案初步定下来，没时间细化的时候，就要出支护和结构的施工图纸，并且现场马上施工。整个项目的设计逻辑全都打乱了，导致在建筑深化的过程中会出现很多问题，修改之前已经出过的图纸，所以大部分时间花在了对接和沟通上面，现在觉得能安安静静画会图已经是幸福了。

说回正题，今天跟大家分享一个之前发现的gh光照分析插件，Sunflower，还是非常好用的，下面跟大家介绍下功能和使用方式。

功能使用说明——Statistical Analysis



虽然这个是一个光照分析软件，但是有一个基本数据分析的工具。这个功能可以让你直观设置建筑的功能分布：


当安装完插件之后，点击Statistical Analysi的按钮，右侧的Panel中会出现属性控制台，可以根据需求进行设置，但是我觉得这个功能并不是很实用，但是看起来还是挺酷的。


功能使用说明——Solar Analysis

这个插件最强大的就是光照分析，我们进行详细的说明。这部分功能主要在Grasshopper中进行实现，方便数据和模型的调整。安装完插件后打开Grasshopper后，可以看到Sunflower的Tab：


其中有两个分析方式，一个是根据建筑物体，判断地面受光点的光照时间——Solar Analysis Plane。


另一个方式是计算建筑物体面上光照时间——Solar Analysis MeshFace：


具体怎么使用我先不多说，根据两个官方案例来介绍使用方式。在下载的安装文件中，有Example文件，其中的GrassHopper文件夹中有两个案例，我们先看第一个“Grid mode for grasshopper”案例，是利用的第一个分析方式。

Grid mode for grasshopper

下图是电池逻辑：


我们一步一步来分析：

1.核心电池

这个电池是Sunflower插件中的电池——Solar Analysis Plane，顾名思义，是计算平面上的光照情况。一共有5个输入端：1.Datatime，输入需要分析的时间端；2.Datatime，输入分析建筑物的经纬度信息，3，Mesh，以Mesh的格式输入建筑模型；4.Grid Point，输入计算光照时间的受光点阵；5.On-Off，输入Boolean值，决定电池是否运行。

接下来我们按输入端进行分析：

2.Datatimes

这个输入端需要通过插件中的电池Datatimes收集需要计算的时间段以及采样间距（Interval），采样间距越小越准确吗，但是计算时间会加长。起始和终止时间需要通过另一个插件电池Datatime输入数据。这部分自由度比较大，但是一般以大寒日或者冬至日进行计算。

3.经纬度设置

这部分也是需要插件电池Location和Location_Coordinate两个电池进行数据的采集和输入。经度和纬度均可以用Location_Coordinate进行设置，最后通过Location，得到经纬度信息。

4.建筑形体生成（Mesh）

这部分输入的是建筑的Mesh模型，这部分利用gh自带的Mesh电池即可。另外这个插件提供了一个Dynamic Selection RhinoObject的电池。这个电池功能类似于gh原生电池中的Pipeline，可以按照图层和物件类型动态拾取犀牛物体：

其实我觉得自带的Pipeline电池更好用^_^。

5.地面受光点设置

这部分是需要输入需要计算范围内的点阵，可以利用GH自带点阵电池。另外插件提供了一个叫Grid的电池，输入范围和间距就可以快速的在一个范围内生成点阵。

6.是否运行

这个就是一个Toggle，不多介绍了

当所有的信息设置完了之后，就可以看到计算结果了：


不同的颜色代表不同的关照时间，整个效果还是很简单易懂的。那么我们再看下电池的输出端的数据：


输出端有计算光照的点坐标，分别按照小时和分钟为单位的时间长度，以及每个点的颜色。这些数据还能做很多事情，比如通过tag电池在每个点标注光照时间的文字并设置相应的文字颜色。逻辑不难，给大家看下电池，就不详细介绍了：



Kazakhstan national library

这个案例采用了第二种分析方式，并进行了一定应用：通过分析建筑物体面上的光照时间，去影响建筑的开窗方式。


电池逻辑如下所示：


整个逻辑以Solar Analysis MeshFace Mode这个电池组为分界线，前面一部分是生成哈萨克斯坦图书馆的形体并转化为Mesh。


后面一部分是根据计算出的光照数据进行开窗。


我们这一期主要介绍这个插件的使用方法，就不过多介绍生成逻辑，下一期会详细分析这个案例。我们主要介绍下计算的过程：


整个计算是由一个核心电池实现的，从名字（Solar Analysis MeshFace）可以看出来，这个电池可以分析Mesh的光照情况。

一共四个输入端，跟上一个的电池Solar Analysis Plane一样，依次输入时间、经纬度、Mesh物体以及Toggle控制开启与否。所以输入端没有什么问题，看下输出端，会输出一个带有颜色属性的Mesh：


这个颜色便于光照时间有关系，颜色越暖，光照时间越长。另外，输出的时间长度可以做更多的事情，比如这个案例中根据光照的开窗，由于案例比较复杂，我们下一期专门继续分析。

插件安装

这个插件可以在Food4Rhino上搜索下载：


直接安装里面的rui文件即可，另外文件夹里面还有常见安装的问题以及案例Example。


如果没有food4rhino有账号，也可以公众号下留言”Sunflower”，即可获得网盘下载链接。如果没有food4rhino有账号，也可以公众号下留言Sunflower，即可获得下载链接。

总结

之前也使用过相关的光照分析软件，Ecotect、HoneyBee之类的，但是都相对比较复杂，这个插件优势是简单，直观，但是又提供了丰富的分析类型和充分的数据内容，满足后续操作的需求。下一期会详细介绍，如何把这个光照分析应用于建筑设计中——哈萨克斯坦图书馆案例。
